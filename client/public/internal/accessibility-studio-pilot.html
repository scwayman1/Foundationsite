Product model

Make the next accessibility decision clear—without pretending automation can make every decision.

The product is not a document editor. It is an orchestration layer: deterministic checks establish evidence, guided review handles judgment, and complex work reaches the right specialist with context intact.

Evidence first

Every finding should identify the rule, location, confidence, and reason it matters—not just produce a score.

Human judgment stays visible

AI can prepare a suggestion; it cannot silently decide whether a description, reading order, or table meaning is adequate.

Fix at the source

When a source DOCX or PowerPoint is available, the system should guide repair there before treating a PDF as the only surface.

Machine checks are necessary. They are not a certification.Live implementation should map findings to adopted institutional standards, retain validator versions, and make residual human-review work explicit.
